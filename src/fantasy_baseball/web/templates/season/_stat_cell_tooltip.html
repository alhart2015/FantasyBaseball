{# Shared stat-cell hover tooltip mechanism. Any element with class `stat-cell` that contains a child `.tooltip` gets a hover tooltip. The tooltip is `position: fixed` and placed by JS (above or below the cell depending on viewport room) so it escapes the `.content` `overflow-y: auto` clip -- a plain CSS `position: absolute` tooltip would be cut off on the first/last rows. `window.bindTooltips()` is idempotent (it removes its own listeners before re-adding) so callers can re-bind after swapping table HTML (e.g. the lineup page's opponent view). Included by lineup.html and roster_audit.html. #}
