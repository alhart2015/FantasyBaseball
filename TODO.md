- [x] **Draft-value metric: realized contribution vs. draft-slot expectation (per player + per team)** _(DONE 2026-07-02, reconciled: PR #153 merged, merge 83da32c; analysis/draft_value.py + scripts/draft_value.py, VAR par curve + skill/luck, loop-review-hardened)_ -- Measure how much each drafted/kept player has actually contributed relative to what a player at their draft slot was *expected* to contribute, then roll it up per team ("who drafted well"). Decisions to settle first:
  - **Metric: SGP vs deltaRoto.** Lean SGP as the primary axis: it's an absolute, context-free per-player value (standings-gain points), so it's comparable across players, teams, and the whole board. deltaRoto is *roster-fit* value (marginal roto pts a player adds to a *specific* team given its category races) -- a different question ("did he help YOUR standings"), where a great player on a category-punting team can show low deltaRoto. So deltaRoto is not the right yardstick for "how good was this pick"; offer it only as an optional secondary per-team lens.
  - **"Expected at draft slot" (the par curve).** At draft time we have preseason projected SGP for every player (the draft board). Sort projected SGP descending -> the value the Nth pick "should" return is roughly the Nth-best projected SGP (a smooth par curve), or more directly the *preseason-projected SGP of the player actually taken at that slot*. Then **draft value = realized SGP(player) - expected SGP(their slot)**: positive = a steal, negative = a bust. Keeper wrinkle (this IS a keeper league): a keeper's "slot" is their keeper cost/round, so keepers are measured against the par value of that round, not a first-overall baseline.
  - **Realized contribution** = season-to-date (or final) SGP from actual stats via the existing `sgp/` machinery, using the SAME denominators/replacement as the draft board so the "expected" and "realized" SGP live on one scale.
  - **Per-team roll-up** = sum of per-player draft values across a team's drafted + kept players -> an "SGP above/below draft expectation" leaderboard.
  - **Reuse:** `sgp/` (SGP + replacement math), the draft board's preseason projected SGP/VAR, the recorded draft results / pick order + keeper costs (`draft_state*.json` / draft dashboard), `data/projections` (the preseason vintage that defines the par curve), and the in-season realized stats produced by the refresh. deltaRoto lives in `scoring.py` if the alternate lens is built.
