# Position-aware draft VAR replacement — design

Date: 2026-06-06
Status: draft (brainstorming) — awaiting user review

## Context / problem

Draft VAR ranks players by `total_sgp - replacement_level[position]`. The
replacement level comes from `calculate_replacement_levels`
(`sgp/replacement.py`): for each position it sorts the eligible pool by
`total_sgp` and takes the Nth-best, where N = league starter slots at that
position.

In an SGP-efficient pool the *marginal* player at every position has roughly
the same total SGP (that is what "marginal" means), so these per-position
floors come out nearly **flat across positions**. VAR then degenerates to
`total_sgp - (about the same constant)` for every hitter, which:

- **over-credits replaceable production** — a 20-SB middle infielder looks
  ~20 SB valuable even though this 10-team league's waiver wire hands out
  ~15 SB free at 2B/SS all season (Nunez ~42 SB, Caballero ~35 SB sat
  un-rostered the entire year); and
- **under-credits scarce production** — catcher, where almost nothing is free
  on waivers, gets the same flat floor as the deep positions.

The empirical fix already exists as **data**: `REPLACEMENT_BY_POSITION`
(`utils/constants.py:335`) is a per-position waiver line calibrated from this
league's own free agents (top-3 un-rostered by SGP, averaged over the
post-draft and current snapshots; generated by
`scripts/analyze_replacement_levels.py`). Its docstring already names this
exact problem ("a streamed middle infielder gives ~15 SB which the flat line
erased"). It is consumed today only by the in-season Monte-Carlo injury
backfill — **not** by the draft VAR path.

This is TODO.md line 63 ("Revisit draft VAR replacement level"). It is an
offseason model-tuning item: it changes `simulate_draft.py` /
`compare_strategies.py` output for *next* year's draft; nothing in-season
moves.

## Goal

Make the draft hitter replacement floor reflect what is actually free on this
league's waiver wire, per position, so VAR stops over-paying for cheap
categories (speed at MI) and stops under-paying for scarce ones (catcher).

## Why a scalar swap is sufficient (the key insight)

SGP is **linear in counting stats** (`stat / denominator`). Therefore
subtracting the *scalar SGP of a realistic per-position waiver stat line* is
algebraically identical to netting each counting category against what is free
at that position:

```
total_sgp(player) - SGP(waiver_line[pos])
  = sum_cat [ sgp_cat(player) - sgp_cat(waiver_line[pos]) ]
```

So a drafted 2B is implicitly charged the ~17 SB already free at 2B, the ~13
HR already free at 2B, etc. No structural rewrite of `calculate_var` is
needed — only the floor it subtracts changes. The cross-position rebalance
(high floor at MI/OF, low floor at C) is the entire fix.

Worked example with the real lines + default denominators
(`R20 HR9 RBI20 SB8 AVG.005`, team_ab 5500, repl_avg .250):

```
C  {55,14,56, 4,107,423} -> SGP  7.65   (cheap to replace -> raises C value)
1B {65,18,68, 6,121,498} -> SGP  9.27
3B {65,17,68, 7,122,496} -> SGP  9.34
2B {62,13,60,17,124,508} -> SGP  9.56   (deep in SB -> docks MI speed)
SS {64,14,62,15,127,520} -> SGP  9.62
OF {65,17,63,15,110,451} -> SGP 10.06   (deepest hitter pool)
```

These are demonstrably **not flat** — the spread is the signal the current
floors erase.

## Approach

**Pure static empirical floor (Option A, user-selected).** For hitter
positions the VAR floor becomes `SGP(REPLACEMENT_BY_POSITION[pos])`, computed
on the same scale as player SGP (same denominators and pool-derived
replacement rates). It is the same value on the static board and on every live
pick — VAR means "value over what's free on waivers," a stable baseline.
In-draft positional scarcity (a run on catchers) stays the job of **VONA**,
the separate scoring mode; we do not fold it back into VAR.

Rejected alternative — hybrid `max(demand_based, empirical)`: because the
demand-based floors are ~flat and the empirical floors are *lower* at C, the
`max` reverts to demand-based exactly at catcher, keeping only the
speed-docking half of the correction and dropping the catcher/scarcity half.
It also re-muddies VAR with draft-timing scarcity that is VONA's job.

## Scope

- **Hitters only.** Pitchers keep the existing demand-based unified-"P" floor,
  unchanged. `REPLACEMENT_BY_POSITION` carries *separate* SP/RP lines, but the
  draft uses generic "P" slots, so collapsing SP+RP into a single empirical "P"
  floor is a distinct modeling question. That belongs to TODO line 61
  (pitcher-VAR overstatement / splitting unified P), which also needs
  end-of-season actuals to validate. Out of scope here.
- **Known effect to measure, not a bug:** recalibrating hitter floors but not
  pitcher floors shifts the *cross-type* board balance (raising most hitter
  floors lowers hitter VAR relative to pitchers). The Phase-0 diagnostic below
  measures the magnitude before we commit; the eventual scoring-mode bake-off
  is where cross-type balance gets judged against real outcomes.

## Components

### 1. New floor function — `sgp/replacement.py`

`position_aware_replacement_levels(pool, starters, denoms, repl_rates) -> dict[str, float]`

- Start from the existing `calculate_replacement_levels(pool, starters)` (so
  the **"P" floor and every demand-based behavior are reused unchanged** — that
  function and its tests stay green).
- Override each hitter position present in both the levels dict and
  `REPLACEMENT_BY_POSITION` (`C,1B,2B,3B,SS,OF`) with
  `calculate_player_sgp(line, denoms=denoms, replacement_avg=repl_rates["avg"], ...)`.
- Override `UTIL` (when present) with the **max** of those empirical hitter
  floors — a UTIL slot is streamed with the best free hitter, which is the
  highest-SGP position line (OF here).
- `IF` stays skipped (handled via positional/UTIL fallback in `calculate_var`),
  exactly as today.
- `calculate_var` is **unchanged** — it just looks up `levels[pos]`.

A small helper computes one line's SGP via the existing
`calculate_player_sgp` (constructing the hitter dict from the constant), so the
floor and player values are produced by the same code path.

### 2. Board wiring — `draft/board.py`

`build_draft_board` already computes `denoms` and pool-derived `repl_rates`
(lines 54/62). Swap the line-75 call from `calculate_replacement_levels(pool,
starters)` to `position_aware_replacement_levels(pool, starters, denoms,
repl_rates)`. Nothing else in the board path changes.

### 3. Recommender wiring — `draft/recommender.py`

`get_recommendations` recomputes floors per pick (line 147). It currently does
**not** compute `denoms`/`repl_rates`, so add them (mirror the board:
`get_sgp_denominators()` + `calculate_replacement_rates(available, starters)`)
and call `position_aware_replacement_levels`. The empirical floors must use the
**same denominators the board's `total_sgp` was built with** so floors and
player values stay on one scale; default denominators match current
`league.yaml` config (the `sgp_overrides` threading gap is TODO line 17 and is
not widened here — noted, not fixed).

VONA is untouched: `calculate_vona_scores` does not consume replacement levels.

### 4. Phase-0 diagnostic — `scripts/diag_draft_replacement.py` (throwaway)

Before wiring, build the real board and print, per position: demand-based floor
vs empirical floor vs delta, plus the VAR before/after for ~8 representative
players (a speed MI, a power corner, a catcher, a balanced OF, a couple
pitchers to show the cross-type tilt). Read-only, writes nothing. This confirms
the direction and sizes the cross-type tilt on real data, and becomes the
before/after evidence for the PR. Deleted (or folded into a test fixture)
before merge.

## Data flow

```
REPLACEMENT_BY_POSITION (constant, per-position waiver stat line)
  -> calculate_player_sgp(line, denoms, repl_rates)     [per hitter position]
  -> position_aware_replacement_levels: demand-based dict
        with hitter positions + UTIL overridden by empirical SGP
  -> calculate_var(player, levels)  [unchanged]
  -> board "var" column / live recommender ranking
```

## Testing

New tests in `tests/test_sgp/test_replacement.py`
(`TestPositionAwareReplacementLevels`):

- **Pitcher floor preserved:** `position_aware_replacement_levels` returns the
  *same* "P" value as `calculate_replacement_levels` on the same pool (we only
  touched hitters).
- **Hitter floors come from the constant:** for a pool + known denoms, each
  hitter floor equals `calculate_player_sgp(REPLACEMENT_BY_POSITION[pos], ...)`
  (assert against the directly-computed scalar, not a magic number).
- **Position spread is real:** assert `C < 1B` and `C < OF` (catcher floor is
  strictly the lowest), pinning the catcher-scarcity / OF-depth ordering so a
  future flat regression fails loudly.
- **UTIL = max of hitter floors** when UTIL slots exist.
- **Empty / tiny pool** still returns a "P" floor (delegates to the existing
  fallback) and the hitter overrides do not raise.

VAR-level test in `tests/test_sgp/test_var.py`: with empirical floors, a
20-SB/low-power MI ranks **below** an equal-total-SGP power corner at the same
position-adjusted comparison (the behavior the fix exists to produce).

Existing `TestReplacementLevels` / `TestFindReplacementPlayers` stay unchanged
and green — the demand-based primitive is not modified.

Full `pytest -v`, `ruff check .`, `ruff format --check .`, `vulture`, and
`mypy` (board.py / replacement.py / var.py are in the mypy set) per the repo
checklist.

## Roadmap — how this fits the bigger picture

This is deliberately the **shared foundation** for the three-way draft scoring
comparison we scoped:

1. **(this spec) position-aware replacement primitive** — improves VAR now and
   is the replacement input the other approaches consume.
2. **deltaRoto draft scoring mode** — promote the existing
   `draft/eroto_recs.py` (`score_roto(team+player) - score_roto(team+replacement)`,
   wraps the in-season `compute_delta_roto`) to a first-class
   `simulate_draft.py` mode, filling all teams' empty slots at replacement.
   That fill, and `eroto_recs._pick_replacement` (TODO line 52), consume the
   same `REPLACEMENT_BY_POSITION` lines this spec wires up — as stat lines
   rather than as scalar SGP.
3. **scoring-mode bake-off** — run VAR / VONA / deltaRoto through
   `compare_strategies.py` / a backtest against actual outcomes to see which
   builds the best team.
4. VONA revisit — deferred.

## Open questions

1. **Pitcher scope** — confirm hitters-only for this increment (recommended),
   deferring empirical pitcher floors + unified-P split to TODO line 61. If the
   Phase-0 diagnostic shows the cross-type tilt is large enough to distort the
   board, we revisit before merge.
2. **Recommender denominators** — compute `repl_rates`/`denoms` live per pick to
   match the board (recommended, negligible cost), vs. accept default
   `replacement_avg` for the floor's small AVG term. Recommend matching the
   board for consistency.
